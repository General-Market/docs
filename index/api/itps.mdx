# ITP Endpoints

An ITP is a basket of assets with a price. These endpoints tell you what that price is, what it was, and what the basket contains. The rest is your decision.

---

## Get ITP Price / NAV

<ParamField query="itp_id" type="string" required>
  The on-chain ITP identifier (hex address, e.g., `0x...`).
</ParamField>

Returns the current NAV per share, priced asset count, and AUM. Tries the data node first; falls back to on-chain if needed.

<CodeGroup>
```bash cURL
curl "https://generalmarket.io/api/itp-price?itp_id=0x1234abcd..."
```

```javascript JavaScript
const res = await fetch(
  "https://generalmarket.io/api/itp-price?itp_id=0x1234abcd..."
);
const data = await res.json();
```

```python Python
import requests

resp = requests.get(
    "https://generalmarket.io/api/itp-price",
    params={"itp_id": "0x1234abcd..."}
)
data = resp.json()
```
</CodeGroup>

<Expandable title="Response">
```json
{
  "itp_id": "0x1234abcd...",
  "nav": "1045000000000000000",
  "nav_display": 1.045,
  "assets_priced": 98,
  "assets_total": 100,
  "aum_usd": 52340.75,
  "total_supply": "50000000000000000000000"
}
```
</Expandable>

| Field | Type | Description |
|-------|------|-------------|
| `itp_id` | string | ITP identifier |
| `nav` | string | NAV per share in wei (18 decimals) |
| `nav_display` | number | Human-readable NAV per share in USD |
| `assets_priced` | number | Number of assets with available price data |
| `assets_total` | number | Total number of assets in the ITP basket |
| `aum_usd` | number | Total assets under management in USD |
| `total_supply` | string | Total ITP shares outstanding in wei |

<Note>
  NAV is computed as `sum(qty[i] * price[i]) / 1e18` where quantities are the fixed per-share amounts stored on-chain. See the pricing model documentation for details.
</Note>

**Dual-source resolution -- the API trusts no single source:**

```
  Request: GET /api/itp-price?itp_id=0x...

  1. Try data-node first (has live API prices, more accurate NAV)
     GET DATA_NODE:8200/itp-price?itp_id=0x...
         |
         +-- success + nav != 0 --> return { source: "data-node", ... }
         |
         +-- fail / nav == 0 ----> fall through

  2. Fallback: read NAV from on-chain Index contract
     eth_call to Index.getITPState(bytes32)
         |
         +-- decode ABI: skip creator, read totalSupply, NAV
         +-- count assets from dynamic array
         +-- return { source: "onchain", ... }

  The "source" field in the response indicates which path was used.
```

## Get NAV Time Series

<ParamField query="itp_id" type="string" required>
  The on-chain ITP identifier.
</ParamField>

<ParamField query="range" type="string" required>
  Time range for the series. One of: `1d`, `7d`, `30d`, `90d`, `1y`.
</ParamField>

NAV over time. The chart tells the story. The numbers are the evidence.

<CodeGroup>
```bash cURL
curl "https://generalmarket.io/api/nav-series?itp_id=0x1234abcd...&range=7d"
```

```javascript JavaScript
const res = await fetch(
  "https://generalmarket.io/api/nav-series?itp_id=0x1234abcd...&range=7d"
);
const series = await res.json();
```

```python Python
import requests

resp = requests.get(
    "https://generalmarket.io/api/nav-series",
    params={"itp_id": "0x1234abcd...", "range": "7d"}
)
series = resp.json()
```
</CodeGroup>

<Expandable title="Response">
```json
[
  { "timestamp": 1708000000, "nav": 1.012 },
  { "timestamp": 1708086400, "nav": 1.018 },
  { "timestamp": 1708172800, "nav": 1.025 },
  { "timestamp": 1708259200, "nav": 1.031 },
  { "timestamp": 1708345600, "nav": 1.038 },
  { "timestamp": 1708432000, "nav": 1.042 },
  { "timestamp": 1708518400, "nav": 1.045 }
]
```
</Expandable>

| Field | Type | Description |
|-------|------|-------------|
| `timestamp` | number | Unix timestamp of the data point |
| `nav` | number | NAV per share at that point in time |

## Get Snapshot

The current state of an ITP -- or all of them. Weights, quantities, NAV, supply.

<ParamField query="itp_id" type="string">
  Optional. When provided, the endpoint returns the snapshot for only the specified ITP.
</ParamField>

<CodeGroup>
```bash cURL
# All ITPs
curl "https://generalmarket.io/api/snapshot"

# Specific ITP
curl "https://generalmarket.io/api/snapshot?itp_id=0x1234abcd..."
```

```javascript JavaScript
// All ITPs
const res = await fetch("https://generalmarket.io/api/snapshot");
const snapshot = await res.json();

// Specific ITP
const res2 = await fetch(
  "https://generalmarket.io/api/snapshot?itp_id=0x1234abcd..."
);
const itpSnapshot = await res2.json();
```

```python Python
import requests

# All ITPs
resp = requests.get("https://generalmarket.io/api/snapshot")
snapshot = resp.json()

# Specific ITP
resp = requests.get(
    "https://generalmarket.io/api/snapshot",
    params={"itp_id": "0x1234abcd..."}
)
itp_snapshot = resp.json()
```
</CodeGroup>

## Get ITP Enrichment

Beyond prices: who founded the projects in this basket, how much venture money they raised, what their TVL looks like, and how the market values them. Four upstream sources, aggregated into one response.

```
GET /api/itp-enrichment?itp_id={itp_id}
```

<ParamField query="itp_id" type="string" required>
  The on-chain ITP identifier (66-character hex string: `0x` + 64 hex chars).
</ParamField>

```
  Data aggregation pipeline:

  +-------------------+     +------------------+     +------------------+
  | Data Node         |     | CoinGecko API    |     | DeFiLlama API    |
  | /snapshot?itp_id= |     | /simple/price    |     | /protocols       |
  +--------+----------+     +--------+---------+     | /raises          |
           |                         |                +--------+---------+
           v                         v                         |
      raw holdings             market data                     v
      (symbol, weight,        (price, mcap,              TVL + funding
       price, name)            24h change)               round data
           |                         |                         |
           +------------+------------+-------------------------+
                        |
                        v
              +---------+----------+
              | Founders Lookup    |
              | (local JSON file)  |
              | age, gender,       |
              | nationality, uni   |
              +--------+-----------+
                       |
                       v
              Combined enrichment response
```

<CodeGroup>
```bash cURL
curl "https://generalmarket.io/api/itp-enrichment?itp_id=0x1234abcd..."
```

```javascript JavaScript
const res = await fetch(
  "/api/itp-enrichment?itp_id=0x1234abcd..."
);
const enrichment = await res.json();
console.log(`${enrichment.holdings.length} holdings`);
console.log(`${enrichment.founders?.total_founders} founders mapped`);
```

```python Python
import requests

resp = requests.get(
    "https://generalmarket.io/api/itp-enrichment",
    params={"itp_id": "0x1234abcd..."}
)
data = resp.json()
for h in data["holdings"][:5]:
    print(f"{h['symbol']:>6s}  {h['weight']:.2%}  ${h.get('price', 0):.2f}")
```

**Holdings fields:**

| Field | Type | Description |
|-------|------|-------------|
| `symbol` | string | Asset ticker |
| `weight` | number | Portfolio weight (0-1) |
| `price` | number | Latest price in USD |
| `name` | string | Human-readable name |
| `image` | string | CoinGecko image URL |
| `coingecko_id` | string | CoinGecko identifier |
| `market_cap` | number | Market capitalization in USD |
| `change_24h` | number | 24-hour price change (%) |

**Founders aggregate** (present when founder data exists for holdings):

| Field | Type | Description |
|-------|------|-------------|
| `total_founders` | number | Total founders matched |
| `total_companies_matched` | number | Holdings with founder data |
| `age_distribution` | array | Age buckets: 20-29, 30-39, 40-49, 50-59, 60+ |
| `gender_split` | array | Gender distribution |
| `top_nationalities` | array | Top 12 nationalities by count |
| `top_universities` | array | Top 10 universities by count |

**DeFi aggregate** (present when holdings have DeFiLlama protocol data):

| Field | Type | Description |
|-------|------|-------------|
| `total_tvl` | number | Combined TVL of matched protocols |
| `avg_tvl_change_7d` | number | Average 7-day TVL change (%) |
| `protocols_with_data` | number | Holdings with DeFiLlama data |
| `top_by_tvl` | array | Top 10 protocols by TVL |

**Funding aggregate** (present when holdings have DeFiLlama raise data):

| Field | Type | Description |
|-------|------|-------------|
| `total_raised_m` | number | Total raised in millions USD |
| `avg_valuation_m` | number | Average valuation in millions USD |
| `total_rounds` | number | Total funding rounds |
| `top_investors` | array | Top 10 investors by deal count |
| `recent_raises` | array | 10 most recent raises |

| Status | Meaning |
|--------|---------|
| `200` | Enrichment data returned (cached 5 min when holdings present) |
| `400` | Invalid `itp_id` format |
| `500` | Enrichment computation failed |

<Note>
  Enrichment results are cached at the CDN for 5 minutes when meaningful data is available. Empty results (no holdings found) are not cached.
</Note>
