# Smart Contract Error Codes

Every error has a code. The code tells you what went wrong. It does not tell you why you tried.

Custom Solidity errors for gas-efficient reverts. All defined in `ErrorsLib.sol`. Format: `EXXX_Name(params)`.

## Order Errors (E001 - E012)

Errors raised during order creation, validation, and lifecycle management.

| Code | Name | Description |
|------|------|-------------|
| `E001` | `OrderBelowMin` | Order amount is below the minimum threshold (0.001 USDC = 1e15 wei with 18 decimals). |
| `E002` | `InsufficientBalance` | User has insufficient USDC balance to cover the order amount. |
| `E003` | `ITPPaused` | The specified ITP is currently paused by governance. |
| `E004` | `SystemPaused` | The entire system is in emergency pause mode. All operations are blocked. |
| `E006` | `ITPNotFound` | The specified ITP identifier does not exist. ITP must be registered before orders can be placed. |
| `E007` | `AssetDelisting` | An asset in this ITP is being delisted. No new orders accepted during delisting. |
| `E008` | `SourceUnavailable` | External liquidity source (AP, DEX) is offline or unreachable. |
| `E009` | `OrderExpired` | Order has expired past its deadline. Orders auto-cancel after 1 hour (default) and the user receives a full refund. |
| `E010` | `FillIncomplete` | Order was only partially filled. The remainder is automatically refunded to the user. |
| `E011` | `InvalidSlippageTier` | Invalid slippage tier specified. Valid tiers: 0 (0.3%), 1 (1%), or 2 (3%). |
| `E012` | `InvalidDeadline` | Order deadline is invalid. Must be in the future and within 24 hours. |

Errors raised during ITP creation, configuration, and weight management.

| Code | Name | Description |
|------|------|-------------|
| `E013` | `WeightBelowMinimum` | ITP weight is below the minimum threshold of 0.25% (2.5e15 with 18 decimals). |
| `E014` | `InvalidWeightSum` | ITP weights do not sum to exactly 100% (1e18). |
| `E015` | `LengthMismatch` | Assets and weights arrays have different lengths. |

| `E017` | `DuplicateAsset` | Duplicate asset address in ITP creation. |

| `E051` | `TooManyAssets` | ITP has too many assets (exceeds maximum). |

## Decoding Errors in the Frontend

The ABI decodes the error. The developer decides what to tell the user.

```typescript
import { decodeErrorResult } from "viem";
import { indexAbi } from "@/lib/contracts/abi";

try {
  await writeContract(/* ... */);
} catch (err) {
  const decoded = decodeErrorResult({
    abi: indexAbi,
    data: err.data,
  });
  // decoded.errorName -> "E002_InsufficientBalance"
  // decoded.args -> [userAddress, requiredAmount, availableAmount]
}
```
