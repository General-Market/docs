# Trading on Vision

Vision lets traders bet on whether asset prices move UP or DOWN over fixed time intervals. This guide covers joining a batch, placing bets, claiming rewards, and withdrawing funds.

## Prerequisites

- A connected wallet with USDC on the Index L3 network
- Navigate to the **Vision** tab on [generalmarket.io](https://generalmarket.io)

## Step 1: Choose a Batch

The Vision page displays all active batches. Each batch card shows:

- **Market count** — number of assets in the batch (e.g., 10 markets)
- **Tick duration** — time between resolutions (e.g., 1 hour)
- **Player count** — current number of active participants
- **TVL** — total USDC deposited across all players

Click a batch card to expand the batch details and access the betting interface.

## Step 2: Place Bets

The betting interface adapts to the number of markets in the batch:

| Market Count | Interface | Description |
|-------------|-----------|-------------|
| 1–20 | Visual (cards) | Click UP or DOWN on each market card |
| 21–100 | Compact (table) | Toggle UP/DOWN in a condensed row layout |
| 100+ | Script (Python) | Write a Python script that returns a list of "UP"/"DOWN" strings |

Each bet is binary: **UP** (price increases) or **DOWN** (price decreases). The resolution type configured for each market determines the threshold for what counts as UP — see [Resolution Types](/reference/resolution-types).

Use the **ALL UP** or **ALL DOWN** buttons to set all markets at once, then adjust individual markets as needed.

## Step 3: Set Deposit and Stake

Before joining, configure two USDC amounts:

- **Deposit amount** — total USDC to deposit into the batch. The deposit funds multiple ticks.
- **Stake per tick** — USDC wagered on each tick. The stake is deducted from the deposit balance every tick.

<Info>
The minimum stake per tick is 0.1 USDC. A deposit of 100 USDC with a stake of 1 USDC per tick funds 100 ticks of play (assuming no winnings or losses).
</Info>

## Step 4: Join the Batch

Clicking **Submit** triggers a three-step process:

### 4a. Bitmap Encoding

The frontend encodes bet selections into a bitmap — a packed binary format where each bit represents one market (1 = UP, 0 = DOWN). The bitmap is hashed with `keccak256` to produce a sealed commitment.

### 4b. On-Chain Commitment

Two wallet transactions execute:

1. **Approve USDC** — authorize the Vision contract to spend the deposit amount (first time only, or if allowance is insufficient)
2. **Join Batch** — call `joinBatch(batchId, depositAmount, stakePerTick, bitmapHash)` on the Vision contract. The USDC is transferred and the bitmap hash is stored on-chain.

### 4c. Bitmap Reveal

After the on-chain commitment confirms, the frontend submits the actual bitmap bytes to all oracle nodes via `POST /p2pool/bitmap`. Each oracle verifies that `keccak256(bitmap)` matches the on-chain commitment before accepting the bitmap.

<Warning>
The bitmap must be revealed to oracles before the next tick resolves. If oracle nodes do not receive the bitmap, the player's bets cannot be scored.
</Warning>

## Step 5: Monitor Results

The expanded batch view shows:

- **Current tick** — the active resolution interval
- **Tick history** — past tick outcomes with per-market results, pool size, and winner/loser counts
- **Player stats** — current balance, total deposited, win rate, and profit/loss

## Step 6: Claim Rewards

After winning ticks resolve, claim accumulated winnings:

1. Click **Claim** on the batch view
2. Select the tick range to claim (e.g., ticks 5–10)
3. The frontend fetches a BLS-signed balance proof from the oracle network
4. Submit the proof on-chain via `claimRewards(batchId, fromTick, toTick, newBalance, blsSignature)`
5. The Vision contract verifies the BLS signature and updates the player's on-chain balance

Claimed USDC is added to the player's available balance and can be used for future ticks or withdrawn.

## Step 7: Deposit More USDC

To add funds to an existing position without leaving the batch:

1. Click **Deposit** on the batch view
2. Enter the additional USDC amount
3. Approve and confirm the `deposit(batchId, amount)` transaction

The additional USDC is added to the player's balance immediately.

## Step 8: Withdraw

To exit a batch entirely:

1. Click **Withdraw** on the batch view
2. The frontend fetches a BLS-signed final balance proof from the oracle network
3. Submit the proof on-chain via `withdraw(batchId, finalBalance, blsSignature)`
4. The Vision contract verifies the BLS signature, transfers the final USDC balance to the player's wallet, and removes the player from the batch

<Info>
Withdrawal requires a BLS balance proof because the player's balance includes off-chain tick resolutions that haven't been claimed yet. The oracle network computes the final balance including all pending winnings and losses.
</Info>

## Updating Bets

To change bet directions without leaving the batch:

1. Adjust UP/DOWN selections in the betting interface
2. Click **Update Bitmap**
3. The frontend encodes a new bitmap, submits a new `keccak256` hash on-chain via `updateBitmap(batchId, newBitmapHash)`, and reveals the new bitmap to oracles

The updated bets take effect starting from the next tick.
