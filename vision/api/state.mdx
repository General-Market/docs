# Batch State

The complete state of a single batch -- every player, every balance, every commitment. A snapshot of collective conviction at one moment in time.

---

## Get Batch State

`GET /vision/batch/{id}/state`

Returns every player's position, balance, and bitmap status. Live data from the in-memory tick scheduler -- the truth, as of right now.

<ParamField path="id" type="number" required>
  The batch ID to query. Must be a valid, existing batch.
</ParamField>

<Tabs>
<Tab title="cURL">
```bash
curl https://generalmarket.io/api/vision/batch/1/state
```
</Tab>
<Tab title="Python">
```python
import requests

batch_id = 1
response = requests.get(f"https://generalmarket.io/api/vision/batch/{batch_id}/state")
state = response.json()

print(f"Batch {state['id']}: {state['player_count']} players")
print(f"Next tick: {state['next_tick']}")
for player in state["players"]:
    print(f"  {player['address']}: balance={player['balance']}, bitmap={player['has_bitmap']}")
```
</Tab>

### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | number | Batch ID |
| `creator` | string | Ethereum address of the batch creator |
| `market_ids` | string[] | Market identifiers in this batch |
| `tick_duration` | number | Seconds between tick resolutions |
| `created_at_tick` | number | The tick number when the batch was created |
| `paused` | boolean | Whether the batch is paused |
| `player_count` | number | Number of active players |
| `next_tick` | number | Unix timestamp of the next tick resolution |
| `players` | object[] | Array of player positions (see below) |

### Player Object

| Field | Type | Description |
|-------|------|-------------|
| `address` | string | Player's Ethereum address |
| `stake_per_tick` | string | Amount staked per tick per market, in wei |
| `balance` | string | Current balance in the batch, in wei |
| `start_tick` | number | The tick number when the player joined |
| `has_bitmap` | boolean | Whether the player has submitted a bitmap for the current tick |

<Warning>
Players with `has_bitmap: false` forfeit their stake without participating in outcomes. To have no opinion is not to be safe -- it is to lose by default.
</Warning>

### Errors

| Status | Condition |
|--------|-----------|
| `404` | Batch with the given ID does not exist |
